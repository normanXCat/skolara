```prisma
model User {
  id              Int       @id @default(autoincrement())
  nom             String
  prenom          String

}

  PARENT
}
```

## 🔑 Système d'authentification

### Stratégie
- **Access Token** : JWT signé, durée 15 minutes, stocké en mémoire côté client (pas localStorage)
- **Refresh Token** : UUID v4, durée 7 jours, stocké en **cookie HttpOnly + Secure + SameSite=Strict**
- **Bcrypt** : hash avec **12 rounds minimum**
- **Rotation des refresh tokens** : à chaque refresh, l'ancien token est révoqué et un nouveau est émis

### Routes API à créer

Respecte **exactement** la structure et le format de réponse des routes API déjà existantes dans le projet.

POST /api/auth/login
POST /api/auth/logout
POST /api/auth/refresh
GET  /api/auth/me

Pour chaque route :
- Validation avec **Zod** (schémas dans un fichier séparé `lib/validations/auth.ts`)
- Gestion d'erreurs typée et centralisée (cohérente avec le reste du projet)
- Format de réponse uniforme `{ success, data?, error?, message? }`
- Codes HTTP appropriés (400, 401, 403, 409, 500)

### Schémas Zod
```typescript
// Valide selon les règles du CDC :
// - Email valide
// - Mot de passe : min 8 chars, 1 majuscule, 1 chiffre
```

### Middleware de protection des routes
Crée `middleware.ts` à la racine :
- Protège toutes les routes `/dashboard/**` et `/admin/**`
- Redirige vers `/login` si non authentifié
- Redirige vers le bon dashboard selon le rôle après login

## 🧪 Tests backend

Crée des tests pour les routes API (utilise le framework de test déjà en place dans le projet) :

- ✅ Login avec credentials valides → retourne access token + cookie refresh
- ❌ Login avec mauvais mot de passe → 401
- ❌ Login avec email inexistant → 401 (même message que mauvais mdp, pas d'énumération)
- ✅ Refresh avec token valide → nouveau access token + rotation du cookie
- ❌ Refresh avec token expiré/révoqué → 401
- ✅ Logout → révoque le refresh token + clear cookie
- ✅ `/me` avec access token valide → retourne les infos user (sans mot de passe)
- ❌ `/me` sans token → 401
